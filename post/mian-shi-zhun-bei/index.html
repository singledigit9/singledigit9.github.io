我最满意的还是《群星守卫》这款手游吧。代码方面，在之前积累的基础上，有所进步。表现效果方面，项目的流程是先由美术设计做效果视频，然后客户端对效果视频进行逆向还原。最终还原的效果基本上能达到效果视频的80%以上。此外，因为有美术效果设计的过于复杂，UI动画还原的效果不好，所以广泛使用了CriWare视频插件。视频插件的封装工作是我负责的，在原来插件的基础上，实现了Intro-Loop模式播放视频，以及视频的切换。

项目中遇到了那些难点，怎么解决的。（4:30）[5min]

项目中遇到的难点就是封装视频插件那块了。插件本身是不开源的，里面由不少代码做了混淆，看不到内部实现机制，只能看到外层的播放器。需求要实现IntroLoop效果，我先是看的官方的Demo，发现官方Demo虽然有切换视频的Demo，但因为插件播放视频需要一个Prepare的过程，所以Demo中两个视频之间切换需要卡几帧，很影响展现效果。然后就想着把第二个视频的Prepare的过程移动到第一个视频播放过程中去。于是采用的第一个方案就是创建不同的视频播放材质球，用直接切换材质球的方式实现视频的切换。但这种方案在材质球切换的时候，有几率会闪材质丢失的哪种粉色粉。为了解决这个问题，进一步研究插件的API，文档，发现有Append的方法可以进行视频拼接。按照这个思路改进，使用一个视频播放器和材质球，发现这种方式的限制很大，首先是视频格式的问题，视频格式要求Intro和Loop一致，且只能用Criware工具压缩的标准格式，h264格式的视频在append的时候有几率失败，而标准格式的体积比较大，而项目中使用视频的量很多，考虑包体积的问题，放弃了这个方案。最后是使用一个材质球，切换多个视频播放器开实现的。在视频播放前，先将Intro和Loop的播放器Prepare，之后Intro完毕后启动Loop进行播放，实现了这个需求。

C#的GC问题

  1. GC步骤
    1. 搜索被引用的托管对象
    2. 确定没有被引用的对象
    3. 释放没有被引用的对象同时回收内存占用
  2. GC是什么：垃圾回收。C#中自动回收不使用的对象。
  3. .Net GC的缺陷
    1. 不能释放所有的资源，只能释放托管的资源
    2. GC并不是实时的，可能会造成性能瓶颈
  4. 什么事托管资源什么事非托管资源
    1. 托管资源就是.Net能够直接回收的，主要是指分配在堆上的资源。托管资源的回收不需要人工干预，自动回收。
    2. 非托管资源就是.Net不知道怎么回收的。Unity里面的例子就比如说贴图，音效等等。需要我们在Unity层面手动管理加载释放的资源。
  5. 可以调用GC.Collect();强制回收
  6. GC是怎么影响性能的
    1. GC是一个后台线程
    2. 当GC进行时，会将当前所有的线程挂起，执行完释放操作后才接着执行正常的程序逻辑。所以频繁GC会导致卡顿。
  7. GC的策略
    1. 标记压缩算法
      1. 从静态变量，静态对象，局部对象，函数调用参数等出发，查找标记所有当前引用的对象。
      2. 总对象已知，存在引用的对象已知，就能知道没有被引用的对象
      3. 将没有引用的对象进行回收
      4. 将不连续的内存压缩成连续的
      5. 修复指针（引用关系）
    2. 分代算法
      1. 分代算法是一个和概率有关的算法
        1. 前提：
          1. 大量创建的对象生命周期比较短，老的对象生命周期比较长
          2. 对部分内存回收，比对所有内存回收要快
          3. 新创建的对象关联程度较强。
        2. 托管堆分配对象内存是连续的。.Net将托管堆分成3个代龄区域，Gen0，Gen1，Gen2
        3. 0代达到阈值，触发0代GC，幸存的进入1代。成本低
        4. 1代达到阈值，触发1代GC，1代GC同时回收0代和1代。幸存的进入2代。成本低
        5. 2代GC将和0代1代2代一起回收。成本一般较高。
        6. 0～2回收频率：100:10:1

算法题

  1. 完全平方数
    1. 动态规划。创建一个n+1的数组
    2. 动态规划公式为：d[i] = min(d[i], d[i - j * j] + 1);
  2. 长度n的数组，有大小为0～n-1的数。其中有重复的，找出任意一个重复数
    1. 字典，hashSet记录
    2. 排序法：如果没有重复的数字，数组第i个数的值应该是i。所以开始遍历。将a[i]的值m，与第m个位置的值a[m]进行比较。如果相等直接返回，如果不等，则交换。之后重复次操作，直至a[i]==i。然后执行下一个for循环。
  3. 圆圈中最后剩下的数字n个数字0～n-1，每m个划掉一个，最后剩下的那个是几。
    1. 递归&&数学推导
    2. f(n, m)表示长度为n的序列，删除n-1次，最后留下元素的序号；
    3. 则f(n-1, m)表示长度为n-1的序列，删除n-2次，最后留下元素的序号；
    4. 我们知道f(n, m)第一次删(m-1)%n，留下以((m-1)%n + 1)%(n-1)开头，长n-1的序列；
    5. 而f(n-1, m)留下元素的序号以0开头，因此两者的序号相差m个距离（别看既%n，又%(n-1)，只是让序号保持在序列长度内，本质上还是右移m个距离，可以循环而已）; f(n, m) = (f(n-1, m) + m) % n
  4. 一串正整数的字符串，移除k个字符，使最终数字最小。
    1. 核心算法，从左到右，如果a[i] < a[i+1]，则移除a[i]
    2. 实现。
      1. 使用一个list，当作栈来使用。
      2. 从左到右开始遍历
        1. for循环list，从后向前循环。
          1. 如果list中的元素大于当前a[i]，则将list中的该元素移除，并且继续循环。
          2. 如果list中的元素小于等于当前a[i]，跳出循环。
        2. 将a[i]压入list
      3. while循环检查，如果list[0]=='0'就将list[0]删掉
      4. 将list转成字符串。
  5. 每隔2k，将k个字符串进行翻转
    1. 要点1，末尾如果补足k个是不是翻转？
    2. 算法
      1. 循环，步进2k
        1. 确定i，j的值。i为当前循环值，j为i+k-1，和length取最小。（i+k是下一组k的第一个，所以这里要减去1）
        2. while循环，i，j头尾互换，判定条件i < j
  6. 找到小镇的法官。法官不相信任何人，其他人都相信法官，法官唯一。现在有二维数组n*2，数组中[a,b]表示a相信b。a，b > 1 , a, b<= n。现在要求找出法官（代表法官的值）
    1. 核心算法。有向图问题，法官的入节点数为n-1，出节点数为0
    2. 建立2个数组。in[n+1],out[n+1]（这里n+1是为了方便计算，不用麻烦的写n-1），用来统计每个人的被信任的人数和信任的人数
    3. 遍历，[a, b]==》out[a]++，in[b]++
    4. 遍历，in out，in[i] == n-1 && out[i] = 0 ==> return i
  7. 求两个矩形的全面积，注意可能有重叠（a,b）（c,d）（e,f）（g,h）
    1. 核心算法，判定判定 min(c, g) > max(a, e) && min(d, h) > max(b, f)，满足这个条件就 有重叠。重叠的面积就是 （min(c, g) - max(a, e)） * （min(d, h)-max(b, f)）
  8. 怎么判断两个矩形相交：知道两个矩形的最小点和最大点坐标
    1. 求出重心坐标
    2. 重心坐标之间相减，取绝对值。
    3. 重心距离x，y，小于矩形长宽和的一般==》有重叠
    4. 重心距离x，y，大于矩形长宽差的一半==》不包含
  9. 字典序排数
    1. 核心算法。本质上是一个十叉树。使用深度优先搜索或者是树的前序遍历
    2. 外部启动，
    3. 深度优先搜索。
      1. 参数，list，存储结果，最大值，当前值，标记位，标记是不是第一层，第一层则不加0

      3. 将当前值加进结果

  10. 随机数索引，一串数组，里面有重复的。输入数组中的一个值，随机返回这个值所在的索引
    1. 核心算法。假设有n个数，每个数，被返回的概率为1/n
      1. 假定现在有n个数，当前选中为第a个数，则a是最终结果的概率为：1/a X (1 - 1/(a+1)) X (1 - 1/(a+2)) X ... X (1 - 1 / n) = 1/n ==>算法概率是正确的。
    2. 算法
      1. 遍历
        1. 如果当前值等于目标值
          1. 加数量
          2. 随机0～（数量-1）
          3. 如果随机值小于1，则选中的索引为当前索引
      2. 返回选中索引
  11. 最大 BST 子树，在给定的树中，找到最大的二叉搜索树，返回这个树的所有节点数量
    1. 使用递归
    2. 一次判定每个节点为root的树是不是二叉搜索树
  12. 翻转游戏 II：两个人依次将“+++++”，中的“++”==》“--”，直至一方不能进行此操作时。即为输掉。
    1. 核心算法：递归
    2. 遍历，查找第一个“++”，将其变为“--”，将新的字符串进行递归。如果结果为false，则第一个人直接获胜，返回true。否则将之前的“--”变回“++”，然后进行下一次遍历。
  13. 输入一个圆形中，能够输出圆形中的随机点
    1. 方法一，大随机，判定不在园内再次随机。空间复杂度O（1），时间复杂度 期望为O（1），最大为O（无穷）
    2. 方法二，随机角度，随机半径。原理是使用极坐标。随机的一个点为x^2 + y^2 = r^2 * random。==》极坐标 = sqrt(random) X r;
