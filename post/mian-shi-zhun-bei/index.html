基础知识

C#的GC问题

  1. GC步骤
    1. 搜索被引用的托管对象
    2. 确定没有被引用的对象
    3. 释放没有被引用的对象同时回收内存占用
  2. GC是什么：垃圾回收。C#中自动回收不使用的对象。
  3. .Net GC的缺陷
    1. 不能释放所有的资源，只能释放托管的资源
    2. GC并不是实时的，可能会造成性能瓶颈
  4. 什么事托管资源什么事非托管资源
    1. 托管资源就是.Net能够直接回收的，主要是指分配在堆上的资源。托管资源的回收不需要人工干预，自动回收。
    2. 非托管资源就是.Net不知道怎么回收的。Unity里面的例子就比如说贴图，音效等等。需要我们在Unity层面手动管理加载释放的资源。
  5. 可以调用GC.Collect();强制回收
  6. GC是怎么影响性能的
    1. GC是一个后台线程
    2. 当GC进行时，会将当前所有的线程挂起，执行完释放操作后才接着执行正常的程序逻辑。所以频繁GC会导致卡顿。
  7. GC的策略
    1. 标记压缩算法
      1. 从静态变量，静态对象，局部对象，函数调用参数等出发，查找标记所有当前引用的对象。
      2. 总对象已知，存在引用的对象已知，就能知道没有被引用的对象
      3. 将没有引用的对象进行回收
      4. 将不连续的内存压缩成连续的
      5. 修复指针（引用关系）
    2. 分代算法
      1. 分代算法是一个和概率有关的算法
        1. 前提：
          1. 大量创建的对象生命周期比较短，老的对象生命周期比较长
          2. 对部分内存回收，比对所有内存回收要快
          3. 新创建的对象关联程度较强。
        2. 托管堆分配对象内存是连续的。.Net将托管堆分成3个代龄区域，Gen0，Gen1，Gen2
        3. 0代达到阈值，触发0代GC，幸存的进入1代。成本低
        4. 1代达到阈值，触发1代GC，1代GC同时回收0代和1代。幸存的进入2代。成本低
        5. 2代GC将和0代1代2代一起回收。成本一般较高。
        6. 0～2回收频率：100:10:1

网络

####TCP和UDP的区别

  1. TCP：
    1. 面向链接
    2. 传输可靠
    3. 适用于传输大量数据
    4. 速度慢
  2. UDP：能够实现服务器同时向多个客户端发送相同的消息
    1. 面向非链接
    2. 传输不可靠
    3. 适用于传输小数据
    4. 速度快
  3. TCP启动前需要3次握手
    1. b向a请求连接
    2. a向b回复可以连接
    3. b向a发送确认连接
  4. TCP断开的4个步骤（ACK确认应答，FIN同步序列号）
    1. A完成传输后，将FIN设为1，提出断开请求
    2. B收到后，将此方向TCP关闭，将ACK设成1
    3. B反向请求关闭，将FIN设为1
    4. A收到后，将ACK设为1，关闭成功。
  5. TCP为什么慢
    1. TCP传输数据，额外的包数据要比UDP大
    2. TCP需要维护连接状态，收发状态等，而UDP不用

值类型，引用类型

  1. 值类型在栈上。引用类型小部分在栈上，保持对堆的引用，其余数据在堆上。
  2. 引用类型需要手动释放或者GC释放，而值类型出了作用域自动回收。

结构体和类

  1. 结构体是值类型，类是引用类型
  2. 结构体无法继承，和被继承。有默认的无参数的构造函数无法隐藏。可以实现接口，但因为接口是引用类型，所以需要注意装箱拆箱。
  3. 结构体适合小的数据存储。
  4. 类适合数据量大，或者内部逻辑复杂的情况。

    2. 将上面概括一下：数据加载到显存；设置渲染状态；调用DrawCall
  2. 几何阶段（GPU）。
  3. 光栅化阶段（GPU）。

GPU中的渲染流水线（几何阶段和光栅化阶段）

  1. 顶点数据
  2. 几何阶段
    1. 顶点着色器：可编程。主要功能是：坐标变换（将模型空间转换到齐次裁剪空间），逐顶点光照，输出后续阶段所需要的数据（这个可选）。
    2. 曲面细分着色器和几何着色器：可编程，但基本上用不到，可以先忽略
    3. 裁剪：可配置
    4. 屏幕映射：不可操作。将齐次裁剪空间映射到屏幕上。（NDC：归一化设备坐标）
  3. 光栅化阶段：计算图元覆盖的像素，计算每个像素的颜色值。
    1. 三角形设置：不可操作。由顶点信息计算出三角形的边界信息
    2. 三角形遍历（也叫扫描变换）：不可操作。检查像素是否被三角形覆盖，被覆盖的话则生成一个片元。（片元并不是像素，它包含了其他诸如深度坐标等信息）

    4. 逐片元操作（也叫输出合并阶段）：可配置。决定片元的可见性（需要一系列测试）；将该片元与颜色缓冲区的颜色进行混合，从而显示最终的颜色。
      1. 流程：模版测试--》深度测试（深度测试--》深度写入测试））--》混合
      2. 上面的测试，通不过的片元就会被丢弃。
      3. 混合需要明确的一个概念：源颜色指的是片元着色器的颜色，目标颜色指的是颜色缓冲区里面的颜色。
  4. 屏幕图像

GPU渲染流水线的一些补充说明

  1. 如果片元没有通过测试，那么对该片元之前的一些操作就浪费掉了。所以往往会将深度测试提前，这种技术叫做Early-Z技术。
  2. GPU会判断片元着色器中的操作是否与提前测试冲突，如果有冲突就会禁用掉提前测试。这就会导致需要处理的片元增多==》性能下降。这也是透明度测试会影响性能的原因。

DrawCall相关

  1. DrawCall影响性能的是CPU而不是GPU。
  2. DrawCall数量太多，就会导致CPU将更多的性能和时间用在提交DrawCall上，从而造成游戏卡顿。
  3. 减少DrawCall的方法。静态物体批处理；减少网格数；避免使用多材质。

协程的实现

利用C#的迭代器来实现的。
在Update中对迭代器进行moveNext，执行到一个yield跳出，并将yield return返回。根据返回的参数，决定是立即进行moveNext操作还是开始计时，等待计时结束了再进行MoveNext操作。每一次MoveNext，都会执行yield前面的代码块。

ECS

分为实体，组件，系统部分。组件为某些指定功能的纯数据，实体包含组件，拥有这个组件就表示实体有这个组件所对应的功能。各个不同的系统，获取拥有指定组件的实体，对他们们进行逻辑操作。
好处一是解耦，新增功能只要新增功能新增对应的组件，和实现对应的system就可以了，实体可以走配置或者编辑器编辑。

算法题

  1. 完全平方数
    1. 动态规划。创建一个n+1的数组
    2. 动态规划公式为：d[i] = min(d[i], d[i - j * j] + 1);
  2. 长度n的数组，有大小为0～n-1的数。其中有重复的，找出任意一个重复数
    1. 字典，hashSet记录
    2. 排序法：如果没有重复的数字，数组第i个数的值应该是i。所以开始遍历。将a[i]的值m，与第m个位置的值a[m]进行比较。如果相等直接返回，如果不等，则交换。之后重复次操作，直至a[i]==i。然后执行下一个for循环。
  3. 圆圈中最后剩下的数字n个数字0～n-1，每m个划掉一个，最后剩下的那个是几。
    1. 递归&&数学推导
    2. f(n, m)表示长度为n的序列，删除n-1次，最后留下元素的序号；
    3. 则f(n-1, m)表示长度为n-1的序列，删除n-2次，最后留下元素的序号；
    4. 我们知道f(n, m)第一次删(m-1)%n，留下以((m-1)%n + 1)%(n-1)开头，长n-1的序列；
    5. 而f(n-1, m)留下元素的序号以0开头，因此两者的序号相差m个距离（别看既%n，又%(n-1)，只是让序号保持在序列长度内，本质上还是右移m个距离，可以循环而已）; f(n, m) = (f(n-1, m) + m) % n
  4. 一串正整数的字符串，移除k个字符，使最终数字最小。
    1. 核心算法，从左到右，如果a[i] < a[i+1]，则移除a[i]
    2. 实现。
      1. 使用一个list，当作栈来使用。
      2. 从左到右开始遍历
        1. for循环list，从后向前循环。
          1. 如果list中的元素大于当前a[i]，则将list中的该元素移除，并且继续循环。
          2. 如果list中的元素小于等于当前a[i]，跳出循环。
        2. 将a[i]压入list
      3. while循环检查，如果list[0]=='0'就将list[0]删掉
      4. 将list转成字符串。
  5. 每隔2k，将k个字符串进行翻转
    1. 要点1，末尾如果补足k个是不是翻转？
    2. 算法
      1. 循环，步进2k
        1. 确定i，j的值。i为当前循环值，j为i+k-1，和length取最小。（i+k是下一组k的第一个，所以这里要减去1）
        2. while循环，i，j头尾互换，判定条件i < j
  6. 找到小镇的法官。法官不相信任何人，其他人都相信法官，法官唯一。现在有二维数组n*2，数组中[a,b]表示a相信b。a，b > 1 , a, b<= n。现在要求找出法官（代表法官的值）
    1. 核心算法。有向图问题，法官的入节点数为n-1，出节点数为0
    2. 建立2个数组。in[n+1],out[n+1]（这里n+1是为了方便计算，不用麻烦的写n-1），用来统计每个人的被信任的人数和信任的人数
    3. 遍历，[a, b]==》out[a]++，in[b]++
    4. 遍历，in out，in[i] == n-1 && out[i] = 0 ==> return i
  7. 求两个矩形的全面积，注意可能有重叠（a,b）（c,d）（e,f）（g,h）
    1. 核心算法，判定判定 min(c, g) > max(a, e) && min(d, h) > max(b, f)，满足这个条件就 有重叠。重叠的面积就是 （min(c, g) - max(a, e)） * （min(d, h)-max(b, f)）
  8. 怎么判断两个矩形相交：知道两个矩形的最小点和最大点坐标
