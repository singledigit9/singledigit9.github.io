命令模式

定义

命令是具现化的方法调用。

个人理解

就是将要执行的具体行为进行封装，交由想执行这个行为的角色执行，而不是直接执行。

例子

要实现一个输入控制系统——比如按下W键前进，S键后退之类的。
首先想到的就是：

if (xxx) action1();
if (yyy) action2();

其中xxx，yyy就是输入满足条件的判断，action1()，action2()就是具体执行的逻辑。

然后要支持玩家改键，那这样写就有些问题了。需要把具体执行的逻辑进行一下封装

public abstract class Command
{
    public abstract void Do();
}

public class MoveUp : Command
{
    public override void Do() { }
}

具体的什么按键执行什么东西封装好了，之后我们需要一个能够获取我们想要的命令对象的工具

public class CommandTool
{
    public Command Get(Type commandType) { }
}

还需要一个根据按键获取对应命令类型的字典

Dictionary<KeyCode, Type> map;

最后改造出来是这样

KeyCode keyCode = Input.GetCurrentPressedKeyCode();
if (keyCode != null)
{
    Command command = CommandTool.Get(map[keyCode]);
    command.Do();
}

再然后想让玩家能够用这套输入控制系统控制不同的角色，那么就需要将玩家要控制的角色作为参数传入进去。

public abstract class Command
{
    public abstract void Do(Avatar avatar);
}

public class MoveUp : Command
{
    public override void Do(Avatar avatar) { }
}

KeyCode keyCode = Input.GetCurrentPressedKeyCode();
if (keyCode != null)
{
    Command command = CommandTool.Get(map[keyCode]);
    command.Do(avatar);
}

到此，命令模式基本完成。

总结

命令模式，就是将具体的行为操作进行封装，从而让目标和目标的行为，以及执行行为的角色进行分离。从而让程序能够在不修改角色代码和执行行为的代码的情况下，不断的扩展角色的行为。
上面的最终例子中，命令由第三方执行，命令中行动的具体角色由第三方指定。从而能够实现一套命令代码，能够应用到这个类型对象的所有角色身上去。
因此，上面的例子不只能够在输入控制系统中使用，AI中也能够使用——只是决定使用哪一个命令的地方不一致罢了。

此外，命令模式还能方便的进行撤销操作。可以在基类中实现撤销的接口，然后其余命令进行实现，就能够完成撤销操作。
如果想连续撤销，可以将当前所执行过的命令使用一个List进行存储，记录当前命令所在的索引值。撤销时，当前命令执行撤销操作，索引值减一，就能够实现。

游戏中另外一个常用的地方就是回放。如果直接将场景中的数据按帧完全存储下来，那么所消耗的内存会相当大。因此可以将场景中每一帧执行了什么命令存储下来，回放时，相当于每帧执行相对应的命令，实现方式和实现效果要好很多。
