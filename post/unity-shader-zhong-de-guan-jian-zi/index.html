Unity Shader中的关键字

2020-04-30 · 10 min read
Unity3D Shader

记录一些Shader的关键字，方便查阅

渲染状态设置选项

  1. Cull 设置剔除模式
    1. Cull Back 剔除背面
    2. Cull Front 剔除正面
    3. Cull Off 关闭剔除
  2. ZTest 设置深度测试时使用的函数
    1. Less Greater
    2. LEqual
    3. GEqual
    4. Equal
    5. NotEqual
    6. Always
  3. ZWrite 深度写入
    1. On 开启
    2. Off 关闭
  4. Blend 设置混合模式
    1. Blend的具体指令内容
    2. BlendOp的具体指令内容

    3. 常见的混合类型
      描述 指令
      正常(normal) Blend SrcAlpha OneMinusSrcAlpha
      柔和相加(sort additive) Blend OneMinusDstColor One
      正片叠底(multiply) Blend DstColor Zero
      变暗(darken) BlendOp Min Blend One One
      变亮(lighten) BlendOp Max Blend One One
      滤色(screen) Blend OneMinusDstColor One
      线性减淡(liner dodge) Blend One One
      等同于 Blend One OneMinusSrcColor
    4. 需要额外注意的是：Max和Min操作，忽略了混合因子；混合默认是加法操作。
  5. ColorMask 颜色掩码：相当于当前Pass最终颜色乘以 (R,G,B,A)。（填了哪个哪个值为1，否则为0）
    1. 0：不允许任何颜色通过
    2. RGB：允许RGB通过
    3. A：只允许A通过
    4. 其他任何RGBA的组合

常用的Tags

  1. Queue：渲染顺序
  2. RenderType：着色器分类，用于着色器替换功能
  3. DisableBatching：禁用批处理
    1. True
    2. False
  4. ForceNoShadowCasting：使用该Shader的物体是否会产生投影
    1. True
    2. False
  5. IgnoreProjector：是否忽略Projector
    1. True
    2. False

Pass中独有的Tags

  1. LightMode：设置Pass在流水线中的角色
  2. RequireOptions：满足条件才渲染这个Pass
    1. SoftVegetion

摄像机和屏幕参数

  1. _WorldSpaceCameraPos：float3，摄像机在世界空间中的位置
  2. _ProjectionParams：float4，摄像机投影参数x=1或者-1，y=Near，z=Far，w=1+1/Far。
  3. _ScreenParams：float4，x=width，y=height，z=1+1/width，w=1+1/height。其中width和height表示摄像机中RenderTarget的像素宽度和高度。
  4. _ZBufferParams：float4，x=1-Far/Near，y=Far/Near，z=x/Far，w=y/Far。用于线性化Z缓存中的深度值。
  5. unity_OrthoParams：float4，x=width，y=height，z没有定义，w=1（正交相机）或者0（透视相机）。其中widht和height为正交投影相机的宽度和高度。
  6. unity_CameraProjection：float4x4，摄像机投影矩阵
  7. unity_CameraWorldClipPlanes[6]：float4[]，摄像机剪裁平面公式，分别是左右，下上，近远。

像素在屏幕中的位置

  1. 语义：WPOS（CG），VPOS（HLSL）

    1. 在400X300的范围内时，使用上面的语义返回的值范围为[0.5,400.5]和[0.5,300.5]，因为OpenGL和DX认为浮点数中间是像素的中心。
    2. z分量为[0,1]，越靠近近裁面越接近0，越靠近远裁面越接近1。
    3. 正交相机下，w始终等于1；透视相机下，w=[1/Near, 1/Far]
    fixed4 frag(float4 sp : WPOS) :SV_Target
{
     return fixed4(sp.xy / _ScreenParams.xy, 0, 1);
}

  2. Unity提供的ComputeScreenPos

    1. 该函数输入为剪裁空间坐标
    2. 这个方式相当于我们手动实现了屏幕映射的过程。
    3. 该函数输出并不是屏幕坐标，而是需要将结果xy除以w分量，才能得到屏幕坐标
    4. 不直接返回屏幕坐标是因为，顶点着色器到片元着色器之间有个插值过程，直接进行除法会破坏插值。因为投影空间不是线性的，线性插值会导致不准确。
    struct vertOut
{
     float4 pos : SV_POSITION;
     float4 scrPos : TEXCOORD0;
}

vertOut vert (appdata_base v)
{
     vertOut o;
     o.pos = mul(UNITY_MATRIX_MVP, v.vertex);
     o.scrPos = CompulateScreenPos(o.pos);

     return o;
}

fixed4 frag(vertOut i) : SV_Target
{
     //最终的屏幕像素坐标
     float2 wcoord = i.scrPos.xy / i.scrPos.w;
     return fixed4(wcrood, 0, 1);
}

代码常用关键字

  1. CGPROGRAM：开始CG代码
  2. ENDCG：CG代码结束
  3. #pragam vertex [name]：预编译指令，定义定点着色器函数
  4. #pragma fragment [name]：预编译指令，定义片元着色器函数
  5. struct：结构体。默认数据类型不满足需求时，可以使用结构体进行定义。
    1. 需要注意的是，一个语义只能存储4个浮点值，如果需要4X4矩阵数据时，可以把矩阵拆成4个float4。
    struct  [structName]
{
    [类型名字] [变量名字] : [语义]
    ……
}
  6. #if UNITY_UV_STARTS_AT_TOP // 判断当前平台是不是DX平台
  7. UNITY_INITIALIZE_OUTPUT([类型], [变量名])：对指定的out结构体进行初始化。
  8. #pragma target 3.0：指定Shader Target。不同的Target的临时寄存器和最大命令数目是不同的。
  9. satureate(x)：将x截取到[0, 1]区间内。如果x是矢量，则对每一个分量进行此操作。
  10. normalize(x)：归一化
  11. UNITY_LIGHTMODEL_AMBIENT.xyz：获取环境光
  12. _WorldSpaceLightPos0.xyz：
    1. 如果是平行光则获取光源方向
    2. 如果是其他光源，则获取光源位置。（可以由光源位置减去顶点位置得到光源方向 ）
  13. dot(a, b)：点积
  14. reflect(i, n)：求反射方向。i为入射方向，n为法线
  15. refract(-normalize(o.worldViewDir), normalize(o.worldNormal), _RefractRatio)可以计算折射，其中_refractRatio为折射率
  16. pow(x, a)：求x的a次方
  17. TRANSFORM_TEX(v.texcoord, _MainTex)：计算最终UV值。
    1. 等价于o.uv = v.texcoord.xy * _MainTex_ST.xy + _MainTex_ST.zw
    2. 原始UV乘以缩放系数再加上UV偏移
  18. cross(a, b)：ab叉乘
  19. UnpackNormal(fixed4 color)：在Texture是Normal map的时候，Unity会根据不同的平台来使用不同的压缩算法。这是rgb不再与xyz相对应了，需要用这个函数将颜色值将颜色值转成法线方向 。具体的可以在UnityCG.cginc中查看。
  20. sqrt(x)：对x开平方
  21. clip(x)：给定参数的任意一个分量有负数，则直接舍弃当前像素的输出颜色。（x可以是float，float2，float3，float4）
    1. 等价于
      void clip(float4 x)
{
    if(any(x < 0))
    {
        discard;
    }
}
  22. #pragma multi_compile_fwdbase：用于前向渲染第一个Pass
  23. #pragma multi_compile_fwdadd：用于前向渲染的第二个Pass
  24. #ifdef USING_DIRECTIONAL_LIGHT：预编译，判断当前光源是不是平行光
  25. #if defined (POINT)：预编译，判断当前光源是不是点光源
  26. #if defined (SPOT)：预编译，判断当前光源是不是 聚光灯。
  27. _LightTexture0：内置的光照衰减纹理
  28. _LightTextureB0：内置的光照衰减纹理（光源使用了cookies情况下）
  29. UnityShader中阴影的三个关键字
  30. SHADOW_COORDS(x)：用于Struct v2f中定义阴影深度纹理。其中x要看我们的TEXCOORD使用了几个，比如我已经用了TEXCOORD0，TEXCOORD1，那么x就写2。
  31. TRANSFER_SHADOW(o)：用于顶点着色器中生成阴影深度纹理
  32. SHADOW_ATTENUATION(i)：用于片元着色器中读取阴影颜色（用来和最终颜色相乘）
  33. 这几个宏对命名有要求，要求a2f的顶点坐标命名必须是v.vertex，v2f中顶点位置变量必须是pos
  34. UNITY_LIGHT_ATTENUATION(atten, i, i.worldPos)：计算光照衰减和阴影。其中atten自动生成，不用额外声明。第二个参数v2f
  35. 立方纹理的采样方法：texCUBE(_TexName, i.xyz)
  36. GrabPass{"_TexName"}：抓屏。抓取的图像存到_TexName中。
    1. 下面的Pass要对应声明sampler2D _TexName; float4 _TexName_TexelSize;
    2. TexName可以省略，但直接声明TexName往往可以提升性能。
    3. _TexName_TexelSize：纹理大小。eg：1/1280，1/720
  37. ComputeGrabScreenPos：与ComputeScreenPos类似，输入为剪裁空间中坐标，输出为屏幕坐标。区别的地方在于抓屏在不同平台上的处理方式不一样
  38. 阴影相关
    1. TRANSFER_SHADOW_CASTER_NORMALOFFSET(v2f o)：顶点着色器中使用
    2. SHADOW_CASTER_FRAGMENT(v2f i) ：片元着色器中使用
  39. 获取使用深度法线纹理相关
    1. C#脚本中的API：camera.depthTextureMode
    2. Shader中声明的贴图：_CameraDepthTexture和_CameraDepthNormalTexture。
  40. LinearEyeDepth：视角空间下进行线性插值
  41. step(a, b)：如果b>a 则返回1，如果b<a则返回0
  42. smothstep(min, max, a)：如果a小于min，返回0；如果a大于max，返回1，否则在0到1之间进行插值。
  43. pragma surface [表面Shader函数名] lightModel [其他可选参数]：声明表面着色器的编译指令。

CGIncludes主要包含的文件以及作用

文件名 描述
UnityCG.cginc 包含最常用的帮助函数，宏，结构体等。
UnityShaderVariables.cginc 编译UnityShader时，自动包含。有许多内置的全局变量，比如 UNITY_MATRIX_MVP 等等。
Lighting.cginc 内置了各种光照模型。使用Surface Shader时自动包含。
HLSLSupport.cginc 编译UnityShader时自动包含。声明了许多跨平台编译的宏和定义。

UnityCG.cginc中常用的结构体和函数

  1. 结构体
名称 描述 包含变量
appdata_base 顶点着色器输入 顶点位置，顶点法线，第一组纹理坐标
appdata_tan 顶点着色器输入 顶点位置，顶点法线，顶点切线，第一组纹理坐标
appdata_full 顶点着色器输入 顶点位置，顶点法线，顶点切线，四组或者更多纹理坐标
appdata_img 顶点着色器输入 顶点位置，第一组纹理坐标
v2f_img 顶点着色器输出 剪裁空间中的位置，纹理坐标
  1. 函数
    下面的函数返回值不保证是单位矢量，使用前注意归一化的问题。
函数名 描述
float3 WorldSpaceViewDir(float4 v) 输入模型空间中顶点位置，返回世界空间中从该点到摄像机的观察方向
float3 UnityWorldSpaceViewDir(float4 v) 输入世界空间中顶点位置，返回世界空间中从该点到摄像机的观察方向
float3 ObjSpaceViewDir(float4 v) 输入模型空间中顶点位置，返回模型空间中从该点到摄像机的观察方向
float3 WorldSpaceLightDir(float4 v) 仅用于前向渲染，输入模型空间中顶点位置，返回世界空间中从该点到光源的光照方向，没有归一化
float3 UnityWorldSpaceLightDir(float4 v) 仅用于前向渲染，输入世界空间中顶点位置，返回世界空间中从该点到光源的光照方向，没有归一化
float3 ObjSpaceLightDir(float4 v) 仅用于前向渲染，输入模型空间中顶点位置，返回模型空间中从该点到光源的光照方向，没有归一化
float3 UnityObjectToWorldNormal(float3 norm) 法线方向从模型空间转到世界空间
float3 UnityObjectToWorldDir(float3 dir) 将一个向量从模型空间转到世界空间
float3 UnityWorldToObjectDir(float3 dir) 将一个向量从世界空间转到模型空间
